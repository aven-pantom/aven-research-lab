# Rejected Path — Ornithopter as Operational Drone Build Recipe

## Why it initially seemed promising

Max asked how we can build an unmanned ornithopter and benefit. The natural pull is to jump directly into airframe instructions, component choices, control tuning, and field deployment.

## Why this path is rejected here

An unmanned aerial vehicle can become dual-use quickly if the research turns into operational guidance for surveillance, evasion, payload integration, or unsafe outdoor deployment.

This repository should preserve the useful research signal without becoming a build recipe for misuse.

## Failed assumption

```text
The most useful next step is to build the flying vehicle immediately.
```

That is probably wrong.

The more useful next step is:

```text
define the lawful civil job → build test harness → collect evidence → prototype safely
```

## Useful remnants

Keep:

- flapping-wing source map;
- event-camera perception angle;
- perching as endurance primitive;
- wing compliance research;
- civil benefit map;
- simulator/benchtop/indoor staged roadmap;
- evidence schema idea.

Do not keep:

- operational flight profiles;
- covert surveillance framing;
- payload guidance;
- evasion tactics;
- deployment recipes.

## Future signal that should reopen the path

Reconsider more specific build work only if it is scoped to a clearly lawful, safe, indoor or educational prototype with explicit constraints, evidence logging, and no harmful deployment use case.
