## Why this felt important

The current AI app-builder frame is too shallow:

```text
prompt → generate app → publish
```

The stronger frame is:

```text
work creates context → agents adapt tools → user keeps the artifact → tools evolve with local data
```

## Useful phrase

**Personal software workshop**

Not an app store, not a prompt box, not a dashboard.

A place where the user’s workbench, files, automations, small apps, components, and agents live together.

## Hidden tension

Users say they want custom software.

But most users do not want to become software maintainers.

So the product has to make customization feel like arranging a workspace, while the agent absorbs maintenance, schema migration, testing, and repair.

## Product-adjacent implication

This could become important for Pantom/Orior-style business systems:

- each client/project gets a local-first workspace;
- agents generate small tools, not massive apps;
- every tool has visible schema, commands, and history;
- artifacts can be exported, forked, and reused;
- maintenance becomes a service layer.
