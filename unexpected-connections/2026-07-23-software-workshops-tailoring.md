# Unexpected Connection — Software Workshops and Tailoring

## Fields connected

- Malleable software
- Local-first tools
- Tailoring / atelier craft
- Small business operations
- Agent-maintained software
- Design systems

## Shared hidden problem

Both software and clothing fail when they assume a universal body.

A prefabricated app is like an off-the-rack suit:

```text
standard sizes
standard assumptions
standard workflow
user adapts body/work to product
```

A workshop/tailor model starts from the person or organization:

```text
measure actual shape
fit around constraints
adjust during use
repair over time
preserve identity through changes
```

## Mechanism transfer

Tailoring has useful primitives software usually lacks:

1. measurement,
2. fitting,
3. alteration,
4. seasonal maintenance,
5. visible seams,
6. repair history,
7. material memory,
8. relationship with a craftsperson.

A personal software workshop could map these to:

```text
measurement → infer schema/process
fitting → generate first working tool
alteration → agent patches behavior/UI
visible seams → inspectable manifest and command history
repair history → tests, migrations, fixes
craftsperson → agent + human operator
```

## Why this matters

Most AI app builders use the wrong metaphor. They promise instant finished software.

But situated tools usually need fitting.

A better product promise:

```text
not “generate any app”
“bring your messy work; leave with a tool fitted to it”
```

## Transferable primitive

**Fitting Session**

A product could formalize a first session:

1. ingest folder/process,
2. identify shape of work,
3. generate rough tool,
4. watch user use it,
5. collect friction,
6. patch tool,
7. record manifest.

The key is not generation speed. It is fit quality.

## Risk of the analogy

Tailoring is high-touch and slow. Software workshops must avoid becoming pure consulting.

The scalable piece is not custom labor. It is the repeatable grammar of fittings, manifests, schemas, patches, and tests.
